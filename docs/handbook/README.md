# Engineering Handbook

How we work. Principles, expectations, and norms — not stack decisions and not "do these exact steps" procedures.

## What lives here

- **Principles and expectations** — values, development flow, career growth, onboarding, meetings.
- One topic per file. Flat structure; no subfolders until a file outgrows itself.

## Index

- [ideas.md](ideas.md) — quotes, concepts and engineering principles that guide judgement calls and thinking
- [development.md](development.md) — how a change moves from idea to production
- [spikes.md](spikes.md) — timeboxed research tickets: when, how, and what "done" looks like
- [incidents.md](incidents.md) — when to declare, SEV1–SEV4 definitions, blameless culture (protocol per [ADR 0040](../adr/0040-incident-management.md))
- [meetings.md](meetings.md) — cadence, sync vs async, what's worth a meeting
- [chat.md](chat.md) — async communications: channels, threads, DMs
- [career.md](career.md) — growth, progression, expectations by level
